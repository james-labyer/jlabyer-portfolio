Overview

Cryptographic Primitives are low-level algorithms that are used as the building blocks of cryptographic protocols for computer systems. The content on this page is sourced from Wikipedia and MDN.

Rationale

Cryptographic primitives are designed to do one task reliably.

In order to be deemed trustworthy, cryptographic primitives are subjected to intense scrutiny from the cryptography community. Developing a new cryptographic primitive requires a lot of time, expertise, and collaboration. As a result, new cryptographic primitives are not developed for specific cryptosystems. Instead, the best cryptographic primitives are used by many cryptosystems.

Any weakness that is discovered in a cryptographic primitive will affect all of the cryptosystems that rely on it. This is why cryptographic primitives are subjected to so much scrutiny.

Combining Cryptographic Primitives

Because each cryptographic primitive is designed for a specific purpose, one cryptographic primitive is usually not enough to provide all of the security features required for a system. In order to provide the desired security features, multiple cryptographic primitives are usually combined together in one carefully designed system.

For example, one cryptographic primitive might be used to encrypt a message, while another might be needed to verify the authenticity of the message.

Commonly Used Primitives

Common types of primitives include:

  • One-way hash functions
  • Symmetric key cryptography
  • Public-key cryptography
  • Digital signature
  • Mix network
  • Private information retrieval
  • Commitment scheme
  • Cryptographically secure pseudorandom number generator
Using Cryptographic Primitives on the Web

Developers of websites and web applications can use the Web Crypto API to access cryptographic primitives that are implemented by the browser according to W3C specification for the Crypto interface.

The Web Crypto API is supported by the major desktop and mobile browsers and by Node.js.

Developers should enlist the help of subject matter experts when considering using cryptographic primitives. An incorrect cryptosystem design can result in security flaws, and they are not always easy to catch. An expert can help ensure that the system is safe.

Commonly Used Web Crypto API Interfaces

Here are some of the most commonly used interfaces in the Web Crypto API. Please see the MDN Web Crypto API documentation for more information.

Create a digital signature:

let signature = await window.crypto.subtle.sign( algorithm, key, data );

Verify a digital signature:

let result = await window.crypto.subtle.verify( algorithm, key, signature, data );

Encrypt data:

let ciphertext = await window.crypto.subtle.encrypt(algorithm, key, data);

Decrypt data:

let plaintext = await window.crypto.subtle.decrypt(algorithm, key, data);

Create a digest of some data:

let digest = await window.crypto.subtle.digest(algorithm, data);
